# Phase 5：运维能力、审计与产品化 UI

## 阶段目标
- 完善可观测性、审计、导出备份和产品级管理界面。

## 本期范围
- 新增审计日志、操作记录、指标、结构化日志、错误告警入口。
- 新增运营总览、Worker 热力图、端口健康图、任务趋势图。
- 完成产品级 UI 美化，提升信息密度与可操作性。

## 非本期范围
- 不引入计费、SaaS 多租户与商业授权体系。

## 核心模块与接口
- `AuditLogService`
- `MetricsService`
- `BackupExportService`
- `OperationsDashboard`

## 数据模型变化
- 新增 `AuditLog`
- 新增 `OperationalMetric`
- 新增 `BackupSnapshot`

## 前后端改动点
- 后端新增审计、导出、指标、备份接口。
- 前端新增运营看板、审计列表、备份导出页和产品化导航。

## 插件扩展点
- `notifier` 插件负责 Webhook、Telegram、邮件、企业 IM 等外部通知。

## 多 Agent 拆分方式
- Agent A：审计与指标。
- Agent B：备份与导出。
- Agent C：产品级 UI 与看板。
- Agent D：通知插件与验收清单。

## 测试计划
- 审计日志记录测试
- 指标暴露测试
- 备份导出测试
- UI 主路径可用性测试

## 阶段验收标准
- 核心操作都有审计记录。
- 可观测性接口可用且字段完整。
- 前端主路径在桌面和移动端可操作。
- 产品界面达到运营台标准，不保留明显模板化痕迹。

## 风险与回滚方式
- 风险：运维能力过早堆叠导致主链路复杂化。
- 回滚：以不影响主链路为前提，运维能力可按模块独立开关。
